Create scripts/scraper/ as a standalone Node.js TypeScript project (separate from the future PWA package.json).

**package.json:** name "sts2-scraper", type "module", scripts: { "scrape": "npx tsx src/scraper.ts", "probe": "npx tsx src/graphql-probe.ts" }. Dependencies: playwright (^1.58.2). DevDependencies: tsx, typescript.

**tsconfig.json:** target ES2022, module NodeNext, moduleResolution NodeNext, strict true, outDir dist.

**src/types.ts:** Define and export:
```typescript
export interface Keyword {
  id: string;          // e.g. "exhaust"
  label: string;       // e.g. "Exhaust"
  category: "mechanic" | "keyword" | "tag";
}

export interface Card {
  id: string;
  name: string;
  cost: number | "X" | null;
  type: "Attack" | "Skill" | "Power" | "Quest" | "Status" | "Curse";
  rarity: "Basic" | "Common" | "Uncommon" | "Rare" | "Special";
  description: string;
  upgraded_description: string;
  characters: string[];   // e.g. ["ironclad"] or ["colorless"] for shared cards
  tier: "S" | "A" | "B" | "C" | "D" | null;
  keywords: Keyword[];
  image_url: string;
}
```

Colorless cards (cards available to all characters or no specific character): use `characters: ["colorless"]` as a sentinel — this is clean for character pool queries (`filter(c => c.characters.includes(selectedChar) || c.characters.includes("colorless"))`).

**src/graphql-probe.ts:** A standalone probe script that:
1. Makes a POST to `https://mobalytics.gg/api/sts2/v1/graphql/query` with Content-Type: application/json and a minimal query body: `{ "query": "{ sts2Cards { id name } }" }` (or introspect with `{ __schema { queryType { name } } }` — try both).
2. Logs the HTTP status code and response body (first 500 chars).
3. Exits 0 if status is 200 with card-shaped data, exits 1 otherwise.
4. Prints a clear summary: "GraphQL endpoint accessible: YES/NO — use direct POST" or "GraphQL requires auth or is blocked — use Playwright fallback".

Run `cd scripts/scraper && npm install && npm run probe` after creating the files. Capture and log the result — this determines which path Task 2 uses.

<task type="auto">
  <name>Task 2: Implement scraper with validation — write public/cards.json</name>
  <files>scripts/scraper/src/scraper.ts, scripts/scraper/src/validate.ts, public/cards.json</files>
  <action>
Implement the full scraper based on the probe result from Task 1. Check probe output to select the path.

**src/validate.ts:** Export `validateCards(cards: unknown[]): Card[]` that:
- Asserts `cards.length >= 400` — throws `Error("Card count ${cards.length} below threshold 400 — aborting write")` if not met
- Asserts every card has non-empty: id, name, description, characters array with at least one entry, image_url
- Asserts cost is a number, "X", or null — not undefined
- Asserts type is one of the known Card type values
- Returns typed Card[] on success
- On any failure: prints the first 3 failing cards for debugging, throws error (never returns partial data)

**src/scraper.ts (GraphQL path — use if probe returned 200 with data):**
- POST to `https://mobalytics.gg/api/sts2/v1/graphql/query` with the full query requesting all fields: id, name, cost, cardType, rarity, description, upgradedDescription, characterClass (or equivalent — inspect actual response shape from probe), tier, keywords, imageUrl
- Map the raw response to the Card schema (normalize field names, cast types)
- Colorless/neutral cards: set `characters: ["colorless"]`
- Call `validateCards(mappedCards)` — if it throws, print error and exit process with code 1 (DO NOT write cards.json)
- Write validated cards to `../../public/cards.json` as a pretty-printed JSON array (indent 2)
- Print final summary: "Wrote N cards to public/cards.json"

**src/scraper.ts (Playwright path — use if GraphQL is blocked):**
- Launch Playwright chromium headless
- Navigate to `https://mobalytics.gg/slay-the-spire-2/wiki/cards`
- Wait for `window.__APOLLO_STATE__` or equivalent serialized state to appear in the page context (use `page.waitForFunction(() => typeof window.__APOLLO_STATE__ !== 'undefined', { timeout: 30000 })`)
- Extract with `page.evaluate(() => JSON.stringify(window.__APOLLO_STATE__))` — parse the Apollo cache and extract card entities (look for keys matching card type pattern)
- Alternatively: intercept the GraphQL network response via `page.on('response', ...)` for requests to `/api/sts2/v1/graphql/query` — this captures the full response without Turnstile interference
- Map extracted data to Card schema, same validation and write logic as GraphQL path
- On Turnstile detection (page title contains "Attention Required" or card count is 0): log error and exit 1 — do NOT write cards.json

**After writing:** Run the scraper: `cd scripts/scraper && npm run scrape`

If the scraper fails (either path), do NOT create a placeholder cards.json. The failure is intentional — it protects against silent bad data. Document the failure mode in a comment at the top of scraper.ts.

Create `public/cards.json` ONLY if the scraper succeeds. If neither path works, create `public/cards.json` as a minimal valid stub with a clear comment: this is a seed file, replace by running the scraper. The stub must have at least 5 real cards hand-authored from the Mobalytics wiki so downstream phases have something to work with.
